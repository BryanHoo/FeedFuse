# 实现全局搜索功能

## Goal

为阅读器增加全局文章搜索能力，支持按关键字模糊搜索历史文章；用户点击结果后，阅读器需要自动切换到对应 RSS 源和文章，并在右栏正文中高亮匹配内容，减少从大量文章中定位目标内容的成本。

## Requirements

- 提供一个阅读器内可直接打开的全局搜索入口
- 支持关键字模糊搜索文章标题、摘要与正文文本
- 搜索结果需要展示足够的上下文，至少包含文章标题、RSS 源、发布时间与命中片段
- 搜索结果标题只展示一行，命中内容预览最多展示两行，保证结果列表更易扫读
- 搜索结果与正文中的关键词高亮需要足够明显，命中词应能被快速识别
- 点击搜索结果后，自动切换 `selectedView` 到目标文章所属 RSS 源
- 点击搜索结果后，自动选中中栏对应文章并渲染右栏正文
- 右栏正文需要高亮当前搜索词的匹配结果
- 搜索流程需要兼容当前三栏阅读器和移动端阅读器，不破坏现有选择与 URL 同步逻辑

## Acceptance Criteria

- [ ] 用户可以在阅读器中输入关键字并看到模糊匹配的文章结果
- [ ] 搜索结果包含跨 RSS 源的文章，而不是只限当前列表快照
- [ ] 搜索结果标题为单行截断，正文命中预览最多两行
- [ ] 点击结果后，左栏切换到目标 RSS 源，中栏选中目标文章，右栏显示对应正文
- [ ] 搜索结果与右栏正文对当前关键字做明显且可见的高亮
- [ ] 搜索为空、无结果、请求失败时有明确 UI 反馈
- [ ] 新增或修改的前后端行为有对应测试覆盖

## Definition of Done

- 新增搜索 API、前端调用与阅读器联动逻辑
- 相关单元测试 / 组件测试通过
- `pnpm lint` 通过
- 相关 `pnpm test:unit` 子集通过

## Technical Approach

新增一个文章搜索 API，基于 `articles` 表的标题、摘要和 HTML 去标签正文做 `ILIKE` 模糊查询，并返回轻量结果集。前端在阅读器中提供搜索弹窗，输入关键字后发起搜索，请求结果列表点击后通过现有 `setSelectedView`、`loadSnapshot` 与 `setSelectedArticle` 串起三栏选择状态。正文高亮通过受控 HTML 标记注入完成，只对当前搜索词做高亮，不引入搜索历史、结果分组或复杂导航。

## Decision (ADR-lite)

**Context**: 当前仓库已有阅读快照与文章详情接口，但没有跨 RSS 源文章搜索能力；仅在前端当前快照内搜索无法满足“全局搜索”要求。正文渲染采用 HTML 注入，必须在不破坏现有链接、图片与翻译内容的前提下处理高亮。

**Decision**: 采用后端搜索 API + 前端搜索弹窗的方案；搜索范围覆盖标题、摘要和正文文本；高亮只在右栏当前显示正文中生效，并基于当前搜索词动态注入 `<mark>` 标记。

**Consequences**: 该方案能满足跨源搜索和精确跳转，但需要新增 API 合约与 HTML 高亮辅助逻辑。为控制复杂度，MVP 不做搜索分页、快捷键面板、命中定位列表与多关键词高级语法。

## Out of Scope

- 搜索历史、最近搜索、保存搜索
- 全局快捷键体系改造
- 搜索结果分页与无限滚动
- 命中位置逐条跳转、上一个/下一个命中导航
- 高级查询语法（AND / OR / 引号精确匹配）

## Technical Notes

- 主要影响层：`src/app/api/`、`src/server/repositories/`、`src/lib/apiClient.ts`、`src/features/reader/`、`src/features/articles/`、`src/store/appStore.ts`
- 现有选择链路由 `setSelectedView`、`loadSnapshot`、`setSelectedArticle` 和 URL 同步逻辑管理，需要复用，不应创建第二套选择状态
- 正文当前由 `ArticleView.tsx` 使用 `dangerouslySetInnerHTML` 渲染，高亮实现需要避免破坏已有图片代理、沉浸式翻译与 AI 摘要相关逻辑
- 搜索结果列表需要保持紧凑可扫读：标题单行、预览双行，并与正文高亮共用一致的强调样式
- 后端规范目录缺失，本次以现有 API route、repository、Zod 校验和测试模式作为实现约束
