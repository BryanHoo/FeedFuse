# Feed 存储上限设计

## 目标

为全局设置增加“每个 feed 最多存储条数”，用户可选固定上限；当某个 feed 的文章总数超过上限时，自动按最旧时间清理超出文章，但始终保留 `isStarred = true` 的文章。

## 范围

- 设置模型新增全局 RSS 存储上限字段
- 设置面板新增对应选择控件
- 设置保存时在上限变化后立即清理现有超量数据
- 后续 RSS 抓取入库和 AI digest 文章生成后都执行同一套上限清理

## 关键规则

- 固定选项：`100 / 200 / 500 / 1000 / 2000`
- 默认值：`500`
- 作用范围：所有 `feed`，包括 `rss` 与 `ai_digest`
- 保护规则：`isStarred = true` 的文章不删除
- 非保护规则：未读文章不保留
- 时间排序：按 `coalesce(published_at, fetched_at), id` 从旧到新删除

## 数据行为

- 目标是尽量将每个 feed 的文章数压到上限以内
- 如果某个 feed 仅收藏文章就已经超过上限，系统不会删除这些收藏文章，因此实际总数可能仍高于配置值

## 实现边界

- 前端只负责配置录入、归一化和保存
- 后端仓库层负责真正的数据清理 SQL
- 设置接口负责“配置改小后立即补清理”
- 入库流程负责“新文章写入后的持续约束”
